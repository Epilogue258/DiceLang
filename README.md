# DiceLang

DiceLang 是一款面向桌面 RPG（跑团）场景的骰子表达式领域特定语言（DSL）。它将骰子投掷、算术运算、条件筛选、变量与宏定义整合为一致的文本表达式，并提供逐步化简的求值过程。

```python
from DiceLang import Interpreter

interp = Interpreter()
results = list(interp("4d6h2 + 1d8 if>5 : 10"))
# 原式: 4D6 + 1D8
# 化简: [6, 6, 1, 1] + 1D8
# 化简: 12 + 1D8
# ...
# 结果: 22
```

## 安装

```bash
git clone https://github.com/your-org/dicelang.git
cd dicelang
uv sync
```

环境依赖：Python ≥ 3.13。

## 快速开始

```python
from DiceLang import Interpreter

interp = Interpreter()

# 基本算术
results = list(interp("1 + 2 * 3"))
results[-1].value  # 7

# 骰子投掷
results = list(interp("3d6"))
results[-1].value  # 如 13

# 骰子 + 选择器：投 4d6 取最高 2 个
results = list(interp("4d6h2"))
results[-1].value  # 如 12

# 变量定义
interp("hp = 100; ac = 15")
results = list(interp("hp - ac"))
results[-1].value  # 85

# 宏定义
interp("&fireball = 8d6")
results = list(interp("&fireballh3"))
```

Interpreter 是可迭代的生成器，每次迭代产出一个 `Result`。每个 `ExprRes` 包含 `.value`（最终值）和 `.steps`（化简过程）。

## 语法规范

```
语句        → 变量定义 | 宏定义 | 表达式
变量定义    → 标识符 (',' 标识符)* 赋值操作符 表达式
宏定义      → '&' 标识符 (',' 标识符)* '=' 表达式
赋值操作符  → '=' | '+=' | '-=' | '*=' | '/=' | '%=' | '^='
表达式      → 项 (('+' | '-') 项)*
项          → 幂 (('*' | '/' | '%') 幂)*
幂          → 原子 ('^' 幂)?
原子        → 数字 | 标识符 | 一元表达式 | 括号表达式 | 函数调用 | 骰子表达式
一元表达式  → ('+' | '-') 原子
括号表达式  → '(' 表达式 ')'
函数调用    → 标识符 '(' 表达式 (',' 表达式)* ')'
骰子表达式  → 原子 'd' 原子 选择器*
选择器      → 'h' 原子 | 'l' 原子 | 'k' | 't'
            | 'if' 条件符 原子 | 'ifc' 条件符 原子 | 'count'
            | ':' 原子
            | 'e' 原子? 条件符? 原子? | '!' 原子? 条件符? 原子?
            | 're' 原子? 条件符? 原子? | 'reroll' 原子? 条件符? 原子?
条件符      → '>' | '<' | '>=' | '<=' | '==' | '!='
```

## 功能详解

### 1. 算术运算

支持加、减、乘、地板除、取模、幂运算，遵循标准运算优先级。

| 运算符 | 说明 | 示例 | 结果 |
|--------|------|------|------|
| `+` | 加 | `3 + 4` | `7` |
| `-` | 减 | `10 - 3` | `7` |
| `*` | 乘 | `5 * 6` | `30` |
| `/` | 地板除 | `7 / 2` | `3` |
| `%` | 取模 | `7 % 3` | `1` |
| `^` | 幂（右结合） | `2^3^2` | `512` |
| `()` | 括号分组 | `(1+2)*3` | `9` |
| `+` | 一元正号 | `+5` | `5` |
| `-` | 一元负号 | `-(3+4)` | `-7` |

### 2. 骰子表达式

基本形式为 `NdM`，投 `N` 个 `M` 面骰，结果求和。骰子参数可以是任意表达式，支持嵌套。

```
3d6          → 投 3 个 6 面骰
2d(4d2)      → 先投 4d2 决定面数，再投 2 个该面数骰
1d20 + 5     → 投 1 个 20 面骰 + 5
```

嵌套骰子可见中间步骤：`3d6d8` → `[6,1,1]D8` → `[掷8d8结果]` → 求和。

### 3. 选择器链——核心特性

选择器链是 DiceLang 最强大的特性，采用管道（pipeline）风格对骰子结果逐项进行标记、筛选、映射，最终输出计算结果。

选择器链附加在骰子表达式之后，从左到右依次执行。每个骰子结果附带一个布尔标记（初始 `False`），选择器通过操作标记与集合实现复杂逻辑。

#### 3.1 标记操作：`hN` / `lN` / `if`

| 选择器 | 说明 |
|--------|------|
| `hN` | 标记当前集合中数值最高的 N 个**未标记**元素 |
| `lN` | 标记当前集合中数值最低的 N 个**未标记**元素 |
| `if >N` | 标记值 **大于** N 的未标记元素 |
| `if <N` | 标记值 **小于** N 的未标记元素 |
| `if >=N` | 标记值 **大于等于** N 的未标记元素 |
| `if <=N` | 标记值 **小于等于** N 的未标记元素 |
| `if ==N` | 标记值 **等于** N 的未标记元素 |
| `if !=N` | 标记值 **不等于** N 的未标记元素 |

N 可以是任意表达式（含骰子），如 `h(1d6)`、`if>(3d6)`。

#### 3.2 删除操作：`k` / `t`

| 选择器 | 说明 |
|--------|------|
| `k` | **保留**已标记的元素，删除未标记的元素，然后清除所有标记 |
| `t` | **丢弃**已标记的元素，保留未标记的元素，然后清除所有标记 |

#### 3.3 计数操作：`count` / `ifc`

| 选择器 | 说明 |
|--------|------|
| `count` | 统计已标记元素的个数（无标记时统计全集合元素数），返回数值 |
| `ifc >N` | `if >N` + `count` 的语法糖 |

#### 3.4 映射操作：`:N`

| 选择器 | 说明 |
|--------|------|
| `:N` | 将当前所有已标记元素的值**替换为 N**，并清除所有标记 |

#### 3.5 爆炸骰：`e` / `!`

| 选择器 | 说明 |
|--------|------|
| `e` | 满值爆炸 1 次（value == sides），`D6!5` 表示爆炸骰 |
| `!` | 同 `e` |
| `eN` | 每颗骰子至多爆 N 次 |
| `e>=N` | 自定义爆炸条件（值 >=N 时爆炸），无次数上限（硬封 10 次） |
| `eN>=M` | 至多 N 次，值 >=M 时爆炸 |

括号表达式支持：`e(1d6)`、`e(3d6)>=5` 等动态参数。

```python
interp("4d6!")          # 满值爆炸
interp("4d6e3>=5")      # 至多 3 次 >=5 爆炸
interp("3d6 e(1d6)")    # 爆炸次数由 1d6 决定
```

#### 3.6 重掷：`re` / `reroll`

| 选择器 | 说明 |
|--------|------|
| `re` | 全重掷 1 次 |
| `reroll` | 同 `re` |
| `reN` | 每颗骰子至多重掷 N 次 |
| `re<N` | 只重掷满足条件的骰子，无上限（10 次） |
| `reN<M` | 至多 N 次，值 <M 时重掷 |

重掷链用 `|` 分隔显示：`4d6re` → `[6|3, 5|2, 2|6, 1|4]`。

```python
interp("4d6re")         # 全重掷一次
interp("4d6re<3")       # 只重掷 <3 的骰子
interp("3d6 re h1")     # 重掷后选最高
```

#### 3.7 最终求和规则

选择器链执行完毕后：
- 若存在已标记元素 → 对**已标记元素**求和
- 若不存在已标记元素 → 对**全集合**求和

### 4. 选择器链示例

以下示例均假设 `4d6` 投出 `[6, 6, 1, 1]`（降序排列）：

**4d6h2**
```
步骤 0: [6, 6, 1, 1]     掷骰
步骤 1: [(6), (6), 1, 1]  h2 标记最高 2 个
步骤 2: 12                对标记项求和：6+6
```

**4d6h2k**
```
步骤 0: [6, 6, 1, 1]     掷骰
步骤 1: [(6), (6), 1, 1]  h2
步骤 2: [6, 6]            k 保留标记项、清空标记
步骤 3: 12                全求和：6+6
```

**4d6 if>4 : 10**
```
步骤 0: [6, 6, 1, 1]     掷骰
步骤 1: [(6), (6), 1, 1]  if>4
步骤 2: [10, 10, 1, 1]    :10 映射为 10
步骤 3: 22                全求和：22
```

**4d6!**
```
步骤 0: [6, 6, 1, 1]     掷骰
步骤 1: [6, D6!3, 6, 1, 1]  满值爆：6→D6!3
步骤 2: 17                全求和：17
```

**4d6re**
```
步骤 0: [6, 6, 1, 1]     掷骰
步骤 1: [6|3, 6|2, 1|2, 1|2]  全重掷一次
步骤 2: 9                 全求和：9
```

### 5. 变量与赋值

```python
# 定义与引用
interp("hp = 50")
interp("hp + 20")    # 70

# 复合赋值
interp("hp = 50")
interp("hp += d6")   # hp 增加 1d6 的投掷结果
interp("hp -= 10")   # hp 减少 10

# 多变量同时赋值
interp("a, b, c = 1, 2, 3")   # 同时赋给 a, b, c
```

支持的复合赋值符：`=` `+=` `-=` `*=` `/=` `%=` `^=`。

### 6. 宏定义

宏是**未求值的 AST 片段**，引用时展开并参与求值。与变量不同，宏保持骰子的随机性——每次引用都会重新投骰。

```python
interp("&gbm = 1d20 + 5")
interp("&gbm")        # 如 18（1d20=13 + 5）
interp("&gbm")        # 如 9（1d20=4 + 5）—— 每次引用重新投骰

# 多名称宏
interp("&atk, &dmg = 1d20, 2d6+3")
```

### 7. 函数调用

```python
interp("max(3d6, 1d20)")   # 取 3d6 和 1d20 的最大值
interp("min(1d8, 5)")      # 取 1d8 和 5 的最小值
interp("max(4d6h2, 1d20)") # 先取 4d6 最高的 2 个求和，再和 1d20 比较
```

## 技术架构

DiceLang 采用三层流水线：

```
输入文本 → [Lexer] → Token 流
         → [Parser] → Statement
         → [Evaluator] → Result
         → [Interpreter] 对外统一入口
```

- **Lexer**：源码 → Token，支持中英文标点
- **Parser**：Pratt Parsing，`parse()` 永不抛异常
- **Evaluator**：逐步化简，`eval()` 永不抛异常
- **Interpreter**：统一入口，管理编译+求值

### 错误体系

```
DiceLangError (Exception)
├── LexerError      # 词法错误
├── ParserError     # 语法错误
├── EvaluatorError  # 求值错误
└── TodoError       # 未实现功能
```

所有错误携带 `pos` / `length` 指向源码位置。
