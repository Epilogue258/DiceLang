# DiceLang

DiceLang 是一款面向桌面 RPG（跑团）场景的骰子表达式领域特定语言（DSL）。它将骰子投掷、算术运算、条件筛选、变量与宏定义整合为一致的文本表达式，并提供逐步化简的求值过程。

```python
from DiceLang import Interpreter

interp = Interpreter()
results = list(interp("4d6h2 + 1d8 if>5 : 10"))
# 原式: 4D6 + 1D8
# 化简: [6, 6, 1, 1] + 1D8
# 化简: 12 + 1D8
# ...
# 结果: 22
```

## 安装

```bash
git clone https://github.com/your-org/dicelang.git
cd dicelang
uv sync
```

环境依赖：Python ≥ 3.13。

## 快速开始

```python
from DiceLang import Interpreter

interp = Interpreter()

# 基本算术
results = list(interp("1 + 2 * 3"))
results[-1].value  # 7

# 骰子投掷
results = list(interp("3d6"))
results[-1].value  # 如 13

# 骰子 + 选择器：投 4d6 取最高 2 个
results = list(interp("4d6h2"))
results[-1].value  # 如 12

# 变量定义
interp("hp = 100; ac = 15")
results = list(interp("hp - ac"))
results[-1].value  # 85

# 宏定义
interp("&fireball = 8d6")
results = list(interp("&fireballh3"))
```

Interpreter 是可迭代的生成器，每次迭代产出一个 `Result`。每个 `ExprRes` 包含 `.value`（最终值）和 `.steps`（化简过程）。

## 语法规范

```
语句        → 变量定义 | 宏定义 | 表达式
变量定义    → 标识符 (',' 标识符)* 赋值操作符 表达式
宏定义      → '&' 标识符 (',' 标识符)* '=' 表达式
赋值操作符  → '=' | '+=' | '-=' | '*=' | '/=' | '%=' | '^='
表达式      → 项 (('+' | '-') 项)*
项          → 幂 (('*' | '/' | '%') 幂)*
幂          → 原子 ('^' 幂)?
原子        → 数字 | 标识符 | 一元表达式 | 括号表达式 | 函数调用 | 骰子表达式
一元表达式  → ('+' | '-') 原子
括号表达式  → '(' 表达式 ')'
函数调用    → 标识符 '(' 表达式 (',' 表达式)* ')'
骰子表达式  → 原子 'd' 原子 选择器*
选择器      → 'h' 原子 | 'l' 原子 | 'k' | 't'
            | 'if' 条件符 原子 | 'ifc' 条件符 原子 | 'count'
            | ':' 原子
条件符      → '>' | '<' | '>=' | '<=' | '==' | '!='
```

## 功能详解

### 1. 算术运算

支持加、减、乘、地板除、取模、幂运算，遵循标准运算优先级。

| 运算符 | 说明 | 示例 | 结果 |
|--------|------|------|------|
| `+` | 加 | `3 + 4` | `7` |
| `-` | 减 | `10 - 3` | `7` |
| `*` | 乘 | `5 * 6` | `30` |
| `/` | 地板除 | `7 / 2` | `3` |
| `%` | 取模 | `7 % 3` | `1` |
| `^` | 幂（右结合） | `2^3^2` | `512` |
| `()` | 括号分组 | `(1+2)*3` | `9` |
| `+` | 一元正号 | `+5` | `5` |
| `-` | 一元负号 | `-(3+4)` | `-7` |

### 2. 骰子表达式

基本形式为 `NdM`，投 `N` 个 `M` 面骰，结果求和。骰子参数可以是任意表达式，支持嵌套。

```
3d6          → 投 3 个 6 面骰
2d(4d2)      → 先投 4d2 决定面数，再投 2 个该面数骰
1d20 + 5     → 投 1 个 20 面骰 + 5
```

### 3. 选择器链——核心特性

选择器链是 DiceLang 最强大的特性，采用管道（pipeline）风格对骰子结果逐项进行标记、筛选、映射，最终输出计算结果。

选择器链附加在骰子表达式之后，从左到右依次执行。每个骰子结果附带一个布尔标记（初始 `False`），选择器通过操作标记与集合实现复杂逻辑。

#### 3.1 标记操作：`hN` / `lN` / `if`

| 选择器 | 说明 |
|--------|------|
| `hN` | 标记当前集合中数值最高的 N 个**未标记**元素 |
| `lN` | 标记当前集合中数值最低的 N 个**未标记**元素 |
| `if >N` | 标记值 **大于** N 的未标记元素 |
| `if <N` | 标记值 **小于** N 的未标记元素 |
| `if >=N` | 标记值 **大于等于** N 的未标记元素 |
| `if <=N` | 标记值 **小于等于** N 的未标记元素 |
| `if ==N` | 标记值 **等于** N 的未标记元素 |
| `if !=N` | 标记值 **不等于** N 的未标记元素 |

标记操作仅改变标记状态，不修改集合大小。若 N 超出可标记元素数量，标记全部可标记元素（不报错）。

#### 3.2 删除操作：`k` / `t`

| 选择器 | 说明 |
|--------|------|
| `k` | **保留**已标记的元素，删除未标记的元素，然后清除所有标记 |
| `t` | **丢弃**已标记的元素，保留未标记的元素，然后清除所有标记 |

`k` 和 `t` 执行后，**所有剩余元素的标记被清空**，后续选择器从干净状态继续工作。

#### 3.3 计数操作：`count` / `ifc`

| 选择器 | 说明 |
|--------|------|
| `count` | 统计已标记元素的个数（无标记时统计全集合元素数），返回数值 |
| `ifc >N` | `if >N` + `count` 的语法糖 |

`count` / `ifc` **必须是选择器链的最后一个**，因为它将骰子结果转换为数值，终结选择器链。

#### 3.4 映射操作：`:N`

| 选择器 | 说明 |
|--------|------|
| `:N` | 将当前所有已标记元素的值**替换为 N**，并清除所有标记 |

映射操作常用于"暴击骰取满"、"重掷变固定值"等场景。

#### 3.5 最终求和规则

选择器链执行完毕后：
- 若存在已标记元素 → 对**已标记元素**求和
- 若不存在已标记元素 → 对**全集合**求和

#### 3.6 选择器链示例

以下示例均假设 `4d6` 投出 `[6, 6, 1, 1]`（降序排列）：

**示例 1：4d6h2**
```
步骤 0: [6, 6, 1, 1]     掷骰结果
步骤 1: [(6), (6), 1, 1]  h2 标记最高 2 个
步骤 2: 12                有标记 → 对标记项求和：6+6
```

**示例 2：4d6h2k**
```
步骤 0: [6, 6, 1, 1]     掷骰结果
步骤 1: [(6), (6), 1, 1]  h2 标记最高 2 个
步骤 2: [6, 6]            k 保留标记项、丢弃未标记项、清空标记
步骤 3: 12                无标记 → 全求和：6+6
```

**示例 3：4d6h2t**
```
步骤 0: [6, 6, 1, 1]     掷骰结果
步骤 1: [(6), (6), 1, 1]  h2 标记最高 2 个
步骤 2: [1, 1]            t 丢弃标记项、保留未标记项、清空标记
步骤 3: 2                 无标记 → 全求和：1+1
```

**示例 4：4d6 if>4 : 10**
```
步骤 0: [6, 6, 1, 1]     掷骰结果
步骤 1: [(6), (6), 1, 1]  if>4 标记大于 4 的元素
步骤 2: [10, 10, 1, 1]    :10 将标记项替换为 10，清空标记
步骤 3: 22                无标记 → 全求和：10+10+1+1
```

**示例 5：4d6 ifc >4**
```
步骤 0: [6, 6, 1, 1]     掷骰结果
步骤 1: [(6), (6), 1, 1]  if>4 标记大于 4 的元素
步骤 2: 2                 count 统计标记元素个数 → 2
```

**示例 6：4d6 l1 t h1 l1**
```
步骤 0: [6, 6, 1, 1]     掷骰结果
步骤 1: [6, 6, 1, (1)]   l1 标记最低 1 个
步骤 2: [6, 6, 1]        t 丢弃标记项
步骤 3: [(6), 6, 1]      h1 标记最高 1 个
步骤 4: [(6), 6, (1)]    l1 标记最低 1 个
步骤 5: 7                 有标记 → 对标记求和：6+1
```

### 4. 变量与赋值

```python
# 定义与引用
interp("hp = 50")
interp("hp + 20")    # 70

# 复合赋值
interp("hp = 50")
interp("hp += d6")   # hp 增加 1d6 的投掷结果
interp("hp -= 10")   # hp 减少 10

# 多变量同时赋值
interp("a, b, c = 1, 2, 3")   # 将三个表达式分别赋给 a, b, c
```

支持的复合赋值符：`=` `+=` `-=` `*=` `/=` `%=` `^=`。

### 5. 宏定义

宏是**未求值的 AST 片段**，引用时展开并参与求值。与变量不同，宏保持骰子的随机性——每次引用都会重新投骰。

```python
interp("&gbm = 1d20 + 5")
interp("&gbm")        # 如 18（1d20=13 + 5）
interp("&gbm")        # 如 9 （1d20=4 + 5）—— 每次引用重新投骰

# 多名称宏
interp("&atk, &dmg = 1d20, 2d6+3")
```

### 6. 函数调用

```python
interp("max(3d6, 1d20)")   # 取 3d6 和 1d20 的最大值
interp("min(1d8, 5)")      # 取 1d8 和 5 的最小值
```

骰子可以先经过选择器再传入函数：

```python
interp("max(4d6h2, 1d20)")   # 先取 4d6 最高的 2 个求和，再和 1d20 比较
```

## 技术架构

DiceLang 采用经典的三层流水线架构：

```
输入文本 → [Lexer] → Token 流
         → [Parser] → Statement
         → [Evaluator] → Result
         → [Interpreter] 对外统一入口
```

- **Lexer（词法分析器）**：将源码字符串转换为 Token 流。支持数字、运算符、骰子符号、选择器关键字、中英文标点、赋值/复合赋值。
- **Parser（语法分析器）**：基于 Pratt Parsing 的递归下降解析器，`parse()` 永不抛异常，返回 `ExprStmt` / `VarDefStmt` / `MacroDefStmt` / `ErrorStmt`。
- **Evaluator（求值器）**：逐步化简 AST，`eval()` 永不抛异常，返回 `ExprRes`（含 value 与 steps）/ `VarDefRes` / `MacroDefRes` / `ErrorRes`。
- **Interpreter（解释器）**：对外的统一入口，管理编译器（Lexer+Parser）和求值器（Evaluator）的交互，支持多语句执行。

### 错误体系

```
DiceLangError (Exception)
├── LexerError      # 词法错误（不可识别字符）
├── ParserError     # 语法错误（括号不匹配、非法运算符）
├── EvaluatorError  # 求值错误（除零、未定义变量、count 非末位）
└── TodoError       # 未实现功能
```

所有错误携带 `pos` / `length` 指向源码位置，供调用方高亮错误。
