# 概述

DiceLang 是用于骰子表达式解析与求值的领域特定语言。本文档定义其核心特性之一：后缀选择器链。该特性允许用户对一次掷骰产生的结果集合进行连续变换（标记、保留、丢弃），并最终输出一个数值。

选择器链采用类似管道（pipeline）的风格，操作符依次作用于当前结果集合，语义清晰且易于扩展。

# 关键字

## 骰子的后缀选择器链

DiceLang 的后缀选择器链提供了一种类似管道的、声明式的骰子结果处理方式。通过四个简单的操作符和清晰的标记-删除-求和流程，用户可以灵活实现“取最高 N 个”、“取最低 N 个”、“保留/丢弃某些值”等复杂逻辑，且边界行为友好（超出数量时取全部，不报错）。该设计易于实现，也易于扩展新的变换操作。

关键字如下:

| 关键字 | 全称    | 说明                                                 |
| ------ | ------- | ---------------------------------------------------- |
| h      | highest | 标记当前集合中数值最高的 N 个元素                    |
| l      | lowest  | 标记当前集合中数值最低的 N 个元素                    |
| k      | keep    | 删除未标记的元素，保留已标记的元素，然后清除所有标记 |
| t      | throw   | 删除已标记的元素，保留未标记的元素，然后清除所有标记 |

注意：h 和 l 仅用于标记，不改变集合大小。k 和 t 会修改集合（删除元素）并强制清除所有标记。

### 执行流程（三个阶段）

阶段 1：掷骰，生成原始集合

掷骰后得到一组数值列表，例如 4d6 可能得到 [6, 4, 3, 2]。每个元素附带一个布尔标记（初始为 False）。

阶段 2：顺序执行选择器链

从左到右依次应用每个操作符，每个操作符的输入是上一个操作符的输出（集合 + 标记状态）。

· hN：在当前集合中，找出数值最高的 N 个未标记元素，将其标记设为 True。若 N 大于未标记元素数量，则标记全部未标记元素（不报错）。
· lN：类似 hN，但取数值最低的 N 个未标记元素进行标记。
· k：删除所有标记为 False 的元素（即保留标记为 True 的元素）。然后清除所有剩余元素的标记（全部设为 False）。
· t：删除所有标记为 True 的元素（即丢弃标记元素）。然后清除所有剩余元素的标记（全部设为 False）。

重要规则：k 和 t 执行后，剩余集合中的所有标记都会被清空。这保证了后续的 h/l 能够正确地从“干净”状态开始标记。

阶段 3：输出结果

· 如果当前集合中存在至少一个标记为 True 的元素，则对这些标记元素的值求和，返回该和。
· 如果当前集合中没有任何标记元素，则对集合中所有元素求和，返回该和。

### 示例输出

示例 1：4d6 掷出 [6,4,3,2]，执行 l1 t h1 l1

| 步骤 | 操作 | 集合状态       | 说明                      |
| ---- | ---- | -------------- | ------------------------- |
| 0    | 初始 | [6, 4, 3, 2]   | 掷骰结果                  |
| 1    | l1   | [6, 4, 3, (2)] | 标记最低的1个(2)          |
| 2    | t    | [6, 4, 3]      | 删除标记的(2)，并清除标记 |
| 3    | h1   | [(6), 4, 3]    | 标记最高的 1 个(6)        |
| 4    | l1   | [(6), 4, (3)]  | 标记最低的 1 个(3)        |
| 5    | 输出 | [6, 3]         | 求和标记项：6 + 3 = 9     |

示例 2：4d6 掷出 [6,4,3,2]，执行 h2 k l2

| 步骤 | 操作 | 集合状态         | 说明                                               |
| :--- | :--- | :--------------- | :------------------------------------------------- |
| 0    | 初始 | [6,4,3,2]        | （无标记）                                         |
| 1    | h2   | [(6), (4), 3, 2] | 标记最高的 2 个（6 和 4）                          |
| 2    | k    | [6, 4]           | 保留标记的（6,4），删除未标记的（3,2），清除标记   |
| 3    | l2   | [(6), (4)]       | 标记最低的 2 个（此时集合只有 2 个元素，全部标记） |
| 4    | 输出 | [6, 4]           | 求和标记项：6 + 4 = 10                             |

示例 3：4d6 掷出 [6,4,3,2]，执行 h2（无 k/t）

| 步骤 | 操作 | 集合状态         | 说明                                                 |
| :--- | :--- | :--------------- | :--------------------------------------------------- |
| 0    | 初始 | [6,4,3,2]        | （无标记）                                           |
| 1    | h2   | [(6), (4), 3, 2] | 标记最高的 2 个                                      |
| 2    | 输出 | [6, 4]           | 求和标记项：6 + 4 = 10; 因为存在标记，所以对标记求和 |

示例 4：仅 k 或 t，无标记时

· [6,4,3,2] k：由于没有标记元素，k 不会删除任何元素，但会清除标记（本来也没有）。最终无标记，输出全部元素之和：6+4+3+2=15。
· [6,4,3,2] t：同理，无标记可删，输出全部元素之和 15。

示例 5：超出数量边界

· 2d6 h3：掷出 [5,3]，h3 尝试标记最高 3 个，但只有 2 个元素 → 标记全部两个。输出 5+3=8。不报错。
· 4d6 h1 k l5：h1 k 后集合变为 [6]（假设最高为 6），然后 l5 标记最低 5 个，实际只有 1 个 → 标记该元素，输出 6。

5. 与函数 max/min 的区分

· 后缀选择器（hN/lN/k/t）作用于骰子结果集合，实现集合变换（取高 N 个、取低 N 个、保留/丢弃标记等）。
· 函数形式（max(...) / min(...)）是数学最大值/最小值函数，接受数值或骰子表达式（先求值为数值），返回数值。

两者可以混合使用，例如 max(4d6h2, 1d20) 中，4d6h2 先通过后缀链计算出一个数值（对标记项求和），再参与 max 比较。
